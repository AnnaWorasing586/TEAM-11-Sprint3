## Sprint 4 Test Plan

### Prototype

**NutriScan AI Web Prototype v1**

Prototype Link:
https://annaworasing586.github.io/TEAM-11-Sprint3/

### Core Flow

อัปโหลดรูปฉลากอาหาร → OCR อ่านข้อมูลโภชนาการ → วิเคราะห์ข้อมูลโภชนาการ → แสดงผลลัพธ์

---

### Test Objective

ประเมินว่าผู้ใช้สามารถใช้งานระบบ NutriScan AI ได้ด้วยตนเองหรือไม่ รวมถึงตรวจสอบว่าผู้ใช้สามารถเข้าใจข้อมูลและผลการวิเคราะห์ที่ระบบแสดงได้มากน้อยเพียงใด

---

### Hypothesis

ทีมเชื่อว่าผู้ใช้จะสามารถอัปโหลดรูปฉลากอาหาร ตรวจสอบข้อมูลที่ระบบ OCR อ่านได้ และเข้าใจผลการวิเคราะห์โภชนาการ เนื่องจากระบบมีขั้นตอนการใช้งานที่เรียบง่ายและไม่ซับซ้อน

---

### Target Users

* นักศึกษามหาวิทยาลัย
* ผู้ที่ซื้ออาหารสำเร็จรูปหรืออาหารบรรจุภัณฑ์เป็นประจำ
* ผู้ที่สนใจด้านสุขภาพและข้อมูลโภชนาการ
* จำนวนผู้ทดสอบ: 5 คน

---

### User Tasks

#### Task 1 — Upload Nutrition Label

อัปโหลดรูปภาพฉลากโภชนาการเข้าสู่ระบบ

#### Task 2 — Review OCR Result

ตรวจสอบข้อมูลโภชนาการที่ระบบ OCR อ่านได้จากรูปภาพ

#### Task 3 — View Nutrition Analysis

ดูผลการวิเคราะห์โภชนาการที่ระบบแสดง และอธิบายสิ่งที่เข้าใจจากผลลัพธ์

---

### Success Criteria

* ผู้ใช้สามารถอัปโหลดรูปภาพได้สำเร็จ
* ผู้ใช้สามารถดูข้อมูลที่ OCR อ่านได้
* ผู้ใช้สามารถเข้าถึงผลการวิเคราะห์โภชนาการได้
* ผู้ใช้สามารถอธิบายผลลัพธ์ที่ได้รับในระดับเบื้องต้น
* ผู้ใช้สามารถใช้งานระบบได้โดยไม่ต้องได้รับความช่วยเหลือมากนัก

---

### Feedback Questions

#### Before Testing

1. คุณเคยอ่านฉลากโภชนาการก่อนซื้ออาหารหรือไม่
2. ปกติคุณใช้ข้อมูลอะไรในการเลือกซื้ออาหาร

#### After Testing

1. ส่วนไหนของระบบที่เข้าใจง่ายที่สุด
2. ส่วนไหนของระบบที่สับสนหรือใช้งานยากที่สุด
3. คุณเข้าใจผลลัพธ์ที่ระบบแสดงหรือไม่
4. หากสามารถปรับปรุงระบบได้ 1 อย่าง คุณอยากให้ปรับปรุงอะไร
5. คุณคิดว่าระบบนี้มีประโยชน์ต่อการเลือกอาหารหรือไม่ เพราะเหตุใด

---

### Data Collection Method

ทีมจะเก็บข้อมูลจาก

* การสังเกตพฤติกรรมระหว่างการใช้งาน
* การบันทึกผลการทำแต่ละ Task
* การจดบันทึกปัญหาที่พบ
* การสัมภาษณ์และรับ Feedback หลังการทดสอบ

---

### Risk / Fallback Plan

* Prototype เปิดใช้งานไม่ได้ → ใช้ Demo Video ประกอบการทดสอบ
* ผู้ทดสอบไม่สามารถเข้าร่วมได้ → เตรียมผู้ทดสอบสำรอง
* อินเทอร์เน็ตมีปัญหา → ใช้ Screenshot และ Local Demo
* รูปภาพตัวอย่างไม่เหมาะสม → เตรียมรูปฉลากอาหารสำรองสำหรับการทดสอบ

---

### Expected Outcome

* ผู้ใช้สามารถดำเนินการตาม Core Flow ได้สำเร็จ
* ทีมได้รับ Feedback เกี่ยวกับความง่ายในการใช้งาน
* ทีมได้รับข้อมูลเกี่ยวกับความถูกต้องของ OCR
* ทีมทราบ Pain Points และแนวทางปรับปรุงระบบใน Sprint ถัดไป
