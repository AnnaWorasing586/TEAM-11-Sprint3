# Sprint 4 Day 1 - Test Plan

## Prototype

NutriScan AI Web Prototype v1

Prototype Link:
https://annaworasing586.github.io/TEAM-11-Sprint3/

Core Flow:
อัปโหลดรูปฉลากอาหาร → OCR อ่านข้อมูล → วิเคราะห์ข้อมูลโภชนาการ → แสดงผลลัพธ์

---

## Test Objective

ต้องการศึกษาว่าผู้ใช้สามารถใช้งานระบบ NutriScan AI ได้ด้วยตนเอง และเข้าใจผลลัพธ์ที่ระบบแสดงหรือไม่

---

## Hypothesis

ทีมเชื่อว่าผู้ใช้จะสามารถอัปโหลดรูปฉลากอาหารและเข้าใจผลการวิเคราะห์ได้ เพราะระบบมีขั้นตอนการใช้งานที่ไม่ซับซ้อน

---

## Target Users

* นิสิตมหาวิทยาลัยที่ซื้ออาหารสำเร็จรูปเป็นประจำ
* ผู้ที่สนใจสุขภาพและข้อมูลโภชนาการ
* จำนวนผู้ทดสอบ 3-5 คน

---

## User Tasks

### Task 1

อัปโหลดรูปฉลากอาหารเข้าสู่ระบบ

### Task 2

ตรวจสอบข้อมูลโภชนาการที่ระบบอ่านได้

### Task 3

ดูผลการวิเคราะห์และอธิบายสิ่งที่เข้าใจจากผลลัพธ์

---

## Feedback Questions

### ก่อนทดสอบ

1. คุณเคยอ่านฉลากโภชนาการก่อนซื้ออาหารหรือไม่
2. ปกติคุณใช้ข้อมูลอะไรในการเลือกซื้ออาหาร

### หลังทดสอบ

1. ส่วนไหนเข้าใจง่ายที่สุด
2. ส่วนไหนสับสนที่สุด
3. คุณเข้าใจผลลัพธ์ที่ระบบแสดงหรือไม่
4. หากปรับปรุงได้ 1 อย่าง คุณอยากให้ปรับอะไร

---

## Risk / Fallback Plan

* Prototype เปิดไม่ได้ → ใช้ Demo Video
* ผู้ใช้ไม่มา → เตรียมผู้ทดสอบสำรอง
* อินเทอร์เน็ตมีปัญหา → ใช้ Screenshot และ Local Demo
