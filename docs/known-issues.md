## Known Issues & Limitations

> Sprint 3 — NutriScan AI (CP5 Submission)
>
> หมายเหตุ: เอกสารนี้จัดทำขึ้น ณ สิ้นสุด Sprint 3 (CP5)
> ก่อนดำเนินการ User Testing กับผู้ใช้จริงใน Sprint 4

---

### Overview

เอกสารนี้สรุปข้อจำกัด (Limitations) และประเด็นที่ยังต้องปรับปรุงของ NutriScan AI Prototype v1 ที่พบระหว่างการพัฒนาใน Sprint 3 เพื่อใช้เป็นข้อมูลสำหรับการวางแผนทดสอบและพัฒนาต่อใน Sprint ถัดไป

---

### Prototype Limitations

#### 1. Web Browser Only

ปัจจุบัน Prototype v1 รองรับการใช้งานผ่าน Web Browser เท่านั้น

ยังไม่มี Mobile Application Version และยังไม่ได้ทดสอบการใช้งานบนอุปกรณ์หลากหลายประเภท

---

#### 2. OCR Accuracy

ระบบ OCR อาจอ่านข้อมูลโภชนาการผิดพลาดในบางกรณี เช่น

- รูปภาพไม่ชัดเจน
- สภาพแสงไม่เพียงพอ
- มีแสงสะท้อนบนฉลากอาหาร
- ตัวอักษรมีขนาดเล็กหรือไม่คมชัด
- รูปแบบฉลากแตกต่างจากชุดข้อมูลที่ใช้ทดสอบ

---

#### 3. Nutrition Analysis Logic

การวิเคราะห์ข้อมูลโภชนาการใน Prototype v1 ยังใช้ Rule-based Logic และเกณฑ์ที่กำหนดไว้ล่วงหน้า

ผลลัพธ์ที่แสดงยังเป็นการสาธิตแนวทางการวิเคราะห์เบื้องต้น และอาจยังไม่ครอบคลุมทุกกรณีของผลิตภัณฑ์อาหารจริง

---

#### 4. Limited Dataset

ระบบยังได้รับการทดสอบกับตัวอย่างฉลากอาหารจำนวนจำกัด

จึงอาจมีข้อจำกัดด้านความแม่นยำเมื่อนำไปใช้งานกับผลิตภัณฑ์หรือรูปแบบฉลากที่หลากหลายมากขึ้น

---

#### 5. User Testing Not Yet Conducted

ณ สิ้นสุด Sprint 3 ระบบยังไม่ได้ผ่านการทดสอบกับผู้ใช้จริง

ดังนั้นทีมยังไม่มีข้อมูลด้าน

- ความง่ายในการใช้งาน (Usability)
- ความเข้าใจของผู้ใช้ต่อผลลัพธ์ที่แสดง
- ความพึงพอใจของผู้ใช้งาน
- ปัญหาที่เกิดขึ้นระหว่างการใช้งานจริง

โดยทีมมีแผนดำเนินการ User Testing ใน Sprint 4 เพื่อรวบรวม Feedback และนำผลไปปรับปรุงระบบต่อไป

---

### Current Workarounds

ในระหว่างการใช้งาน Prototype v1 สามารถลดผลกระทบจากข้อจำกัดข้างต้นได้โดย

- ใช้รูปภาพฉลากอาหารที่มีความคมชัดสูง
- ถ่ายภาพในบริเวณที่มีแสงสว่างเพียงพอ
- หลีกเลี่ยงแสงสะท้อนบนฉลากอาหาร
- ตรวจสอบผลลัพธ์ OCR ก่อนนำข้อมูลไปใช้งาน
- ทดสอบด้วยฉลากอาหารที่มีรูปแบบมาตรฐาน

---

### Planned Actions for Sprint 4

ทีมมีแผนดำเนินการดังต่อไปนี้ใน Sprint 4

1. ทดสอบระบบกับผู้ใช้จริงจำนวน 5–10 คน
2. เก็บ Feedback และ Observation จากการใช้งาน
3. ประเมินความแม่นยำของ OCR จากสถานการณ์จริง
4. ปรับปรุง User Experience และการแสดงผลข้อมูล
5. จัดลำดับความสำคัญของปัญหาเพื่อวางแผนพัฒนา Sprint ถัดไป

---

### Conclusion

NutriScan AI Prototype v1 สามารถทำงานตาม Core Flow หลักได้สำเร็จในระดับ Prototype ได้แก่ การอัปโหลดรูปภาพ การอ่านข้อมูลด้วย OCR และการแสดงผลการวิเคราะห์โภชนาการ อย่างไรก็ตาม ยังมีข้อจำกัดหลายด้านที่ต้องได้รับการตรวจสอบและปรับปรุงเพิ่มเติมผ่านการทดสอบกับผู้ใช้จริงใน Sprint 4
