## Prototype v2 Plan

> Sprint 5 Planning
>
> NutriScan AI (Team 11)

เอกสารนี้สรุปแผนการพัฒนา Prototype v2 โดยอ้างอิงจากผลการทดสอบผู้ใช้ (User Testing) ใน Sprint 4 รวมถึง [insights](insights.md) และ [Priority Fix List](priority-fix-list.md) ที่ทีมได้วิเคราะห์ไว้

---

### Current Version (Prototype v1)

ฟังก์ชันที่มีอยู่ในปัจจุบัน

- อัปโหลดรูปฉลากโภชนาการ
- OCR อ่านข้อมูลจากรูปภาพ
- วิเคราะห์ข้อมูลโภชนาการ
- แสดงผลลัพธ์ผ่านหน้าเว็บ

Core Flow:

```text
Upload Image

↓

OCR Processing

↓

Nutrition Analysis

↓

Result Display
```

---

### Prototype v2 Goals

ทีมต้องการปรับปรุงความถูกต้องของข้อมูลและประสบการณ์การใช้งาน โดยยังคง Core Flow เดิมไว้

---

### Planned Improvements

#### 1. Improve OCR Accuracy

- ปรับปรุงการอ่านข้อมูลจากฉลากอาหาร
- ลดข้อผิดพลาดในการอ่านตัวเลขและข้อความ
- เพิ่มการทดสอบกับฉลากหลายรูปแบบ

---

#### 2. Add Image Upload Guidance

- เพิ่มคำแนะนำก่อนอัปโหลดรูปภาพ
- แนะนำวิธีถ่ายภาพให้เห็นฉลากชัดเจน
- ลดปัญหาที่เกิดจากรูปภาพคุณภาพต่ำ

---

#### 3. Improve Nutrition Analysis Explanation

- เพิ่มคำอธิบายผลการวิเคราะห์
- เพิ่มข้อความที่เข้าใจง่ายสำหรับผู้ใช้ทั่วไป
- ช่วยให้ผู้ใช้สามารถตีความข้อมูลโภชนาการได้ดีขึ้น

---

#### 4. Improve User Interface

- ปรับปรุงการจัดวางข้อมูล
- เพิ่มความชัดเจนของผลลัพธ์
- ปรับปรุงประสบการณ์การใช้งานโดยรวม

---

### Expected Outcome

Prototype v2 ควรสามารถ

- อ่านข้อมูลได้แม่นยำขึ้น
- ลดข้อผิดพลาดจาก OCR
- ช่วยให้ผู้ใช้เข้าใจผลลัพธ์ได้ง่ายขึ้น
- มอบประสบการณ์การใช้งานที่ดีขึ้น

---

### Scope for Sprint 5

ทีมจะมุ่งเน้นการปรับปรุงคุณภาพของระบบเดิม มากกว่าการเพิ่มฟีเจอร์ใหม่ เพื่อให้ NutriScan AI มีความเสถียรและพร้อมสำหรับการทดสอบเพิ่มเติมในอนาคต
